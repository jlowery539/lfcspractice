Section 1: Configure and Manage LVM Storage

Labs 1–20 · Physical Volumes, Volume Groups, Logical Volumes, Snapshots, Thin Provisioning

Lab 1: Identify Block Devices and Partitions

Use standard Linux tools to enumerate all attached block devices.

  1. Run lsblk and review output columns (NAME, MAJ:MIN, RM, SIZE, RO, TYPE, MOUNTPOINT)
  2. Run blkid to display UUIDs and filesystem types of all partitions
  3. Run fdisk -l to list disk geometry, sector sizes, and partition tables
  4. Run cat /proc/partitions to view kernel-reported partitions
  5. Compare outputs and confirm all four virtual disks (/dev/sdb–/dev/sde) are listed
All four virtual disks appear in every command's output. No partitions exist on new disks.
lsblk -f adds filesystem and UUID columns — very useful for quick overview.

Lab 2: Create Partitions with fdisk

Use fdisk to create two 2 GB partitions on /dev/sdb, set type to Linux LVM.

  1. Run fdisk /dev/sdb to enter interactive partition editor
  2. Create new partition (n), primary, partition 1, size +2G
  3. Change type (t) to 8e (Linux LVM)
  4. Repeat for second 2 GB partition (partition 2)
  5. Write (w) and exit
lsblk /dev/sdb shows two 2 GB partitions. fdisk -l shows type "Linux LVM".

Lab 3: Create Partitions with gdisk (GPT)

Use gdisk to create GPT partition table on /dev/sdc with one 3 GB partition type 8e00.

  1. Run gdisk /dev/sdc to open GPT partition editor
  2. New partition (n), accept defaults, size +3G
  3. Type code 8e00
  4. Write (w) and confirm
gdisk -l /dev/sdc shows GPT table with 3 GB partition type 8E00. lsblk confirms.
fdisk uses MBR, gdisk uses GPT. GPT supports disks >2 TB and >4 primary partitions.

Lab 4: Initialize Physical Volumes

Initialize partitions as LVM Physical Volumes using pvcreate.

  1. Run pvcreate /dev/sdb1 /dev/sdb2 /dev/sdc1
  2. Verify with pvs
  3. Inspect details: pvdisplay /dev/sdb1
  4. Scan for all PVs: pvscan
pvs lists all three PVs. pvdisplay shows PE size (default 4 MB).

Lab 5: Create a Volume Group

Create VG "vg_data" from two Physical Volumes.

  1. Run vgcreate vg_data /dev/sdb1 /dev/sdb2
  2. Verify with vgs
  3. Inspect details: vgdisplay vg_data
vgs shows vg_data with 2 PVs and ~4 GB total.

Lab 6: Create Logical Volumes

Create two LVs within vg_data.

  1. Run lvcreate -n lv_projects -L 1G vg_data
  2. Run lvcreate -n lv_backups -L 500M vg_data
  3. Verify with lvs
  4. Inspect details: lvdisplay
lvs shows both LVs with correct sizes.

Lab 7: Create Filesystems on Logical Volumes

Format LVs with different filesystem types and mount them.

  1. Run mkfs.ext4 /dev/vg_data/lv_projects
  2. Run mkfs.xfs /dev/vg_data/lv_backups
  3. Create mount points: mkdir -p /mnt/projects /mnt/backups
  4. Mount both: mount /dev/vg_data/lv_projects /mnt/projects and mount /dev/vg_data/lv_backups /mnt/backups
df -Th shows correct filesystem types and sizes.

Lab 8: Extend a Volume Group

Add new PV to existing VG using vgextend.

  1. Confirm /dev/sdc1 is a PV: pvs
  2. Run vgextend vg_data /dev/sdc1
  3. Verify new size: vgs
vgs shows 3 PVs and ~7 GB total.

Lab 9: Extend a Logical Volume (ext4)

Extend LV with ext4 and resize filesystem.

  1. Run lvextend -L +500M /dev/vg_data/lv_projects
  2. Resize filesystem: resize2fs /dev/vg_data/lv_projects
lvs shows 1.5 GB. df -Th reflects new size.

Lab 10: Extend a Logical Volume (XFS)

Extend LV with XFS and grow filesystem.

  1. Run lvextend -L +300M /dev/vg_data/lv_backups
  2. Grow XFS: xfs_growfs /mnt/backups (takes mount point, not device)
df -Th shows 800 MB.
resize2fs takes device path, xfs_growfs takes mount point. This is a common exam question.

Lab 11: Extend LV and Filesystem in One Command

Use the -r flag for combined extend and resize.

  1. Run lvextend -r -L +200M /dev/vg_data/lv_projects
  2. Observe combined output showing both LV extend and filesystem resize
df -Th shows ~1.7 GB.
The -r (--resizefs) flag works with both ext4 and XFS. Always prefer this on the exam for speed.

Lab 12: Reduce a Logical Volume (ext4)

Shrink ext4 filesystem and LV. Understand that XFS cannot be shrunk.

  1. Unmount: umount /mnt/projects
  2. Force check: e2fsck -f /dev/vg_data/lv_projects
  3. Shrink filesystem: resize2fs /dev/vg_data/lv_projects 1G
  4. Reduce LV: lvreduce -L 1G /dev/vg_data/lv_projects
  5. Remount: mount /dev/vg_data/lv_projects /mnt/projects
df -Th shows ~1 GB.
XFS cannot be shrunk — ever. Always shrink the filesystem BEFORE reducing the LV for ext4, or data loss will occur.

Lab 13: Create LV Using Percentage of Free Space

Use percentage-based allocation with lowercase -l flag.

  1. Check free space: vgs
  2. Run lvcreate -n lv_logs -l 50%FREE vg_data
  3. Format: mkfs.ext4 /dev/vg_data/lv_logs
  4. Mount: mount /dev/vg_data/lv_logs /mnt/logs
lvs shows ~half the previously free space.
Other options: 100%FREE, 50%VG, 100%PVS. Note lowercase -l for extents/percentages vs uppercase -L for absolute sizes.

Lab 14: Remove a Logical Volume

Safely remove an LV following proper procedure.

  1. Unmount: umount /mnt/logs
  2. Remove: lvremove /dev/vg_data/lv_logs, confirm with y
lvs no longer shows lv_logs. vgs shows increased free space.

Lab 15: Remove a Physical Volume from a VG

Migrate data off PV and remove from VG.

  1. Check usage: pvs -o+pv_used
  2. Migrate data: pvmove /dev/sdc1
  3. Remove from VG: vgreduce vg_data /dev/sdc1
  4. Remove PV label: pvremove /dev/sdc1
vgs shows 2 PVs. pvs no longer lists /dev/sdc1.
pvmove can take time on large volumes. Ensure remaining PVs have enough free space.

Lab 16: Rename Volume Groups and Logical Volumes

Rename VGs and LVs, update configuration.

  1. Rename VG: vgrename vg_data vg_storage
  2. Rename LV: lvrename vg_storage lv_projects lv_work
  3. Update /etc/fstab with new VG/LV names
  4. Remount: mount -a
vgs shows vg_storage. lvs shows lv_work. mount -a succeeds.

Lab 17: Create an LVM Snapshot

Create, use, and remove an LVM snapshot.

  1. Write test data to /mnt/projects/testfile.txt
  2. Create snapshot: lvcreate -s -n lv_work_snap -L 256M /dev/vg_storage/lv_work
  3. Modify original data on /mnt/projects
  4. Mount snapshot read-only: mount -o ro /dev/vg_storage/lv_work_snap /mnt/snap
  5. Verify original data preserved in snapshot
  6. Unmount and remove: umount /mnt/snap && lvremove /dev/vg_storage/lv_work_snap
Snapshot preserves original data. After removal, lvs no longer shows snapshot.

Lab 18: Create a Thin Provisioned Pool and Volume

Create thin pool and overcommitted thin LV.

  1. Create thin pool: lvcreate -T -n thin_pool -L 1G vg_storage
  2. Create thin volume: lvcreate -T -n lv_thin -V 2G vg_storage/thin_pool
  3. Format and mount: mkfs.ext4 /dev/vg_storage/lv_thin
  4. Observe with lvs
lvs shows 2 GB virtual size backed by 1 GB pool. df reports 2 GB.
Thin provisioning allows overcommitting. If pool fills, filesystem goes read-only. Monitor usage carefully.

Lab 19: Set a Custom Physical Extent Size

Create VG with non-default PE size.

  1. Initialize PV: pvcreate /dev/sdd
  2. Create VG with 16M PE: vgcreate -s 16M vg_custom /dev/sdd
  3. Create LV using extents: lvcreate -n lv_extents -l 32 vg_custom (32 × 16 MB = 512 MB)
vgdisplay shows PE Size 16.00 MiB. lvs shows 512 MB.

Lab 20: Complete LVM Teardown and Rebuild

Systematic teardown and rebuild to reinforce the full LVM workflow.

  1. Unmount all LVM filesystems
  2. Remove all LVs: lvremove
  3. Remove all VGs: vgremove
  4. Remove all PVs: pvremove
  5. Verify clean state with pvs, vgs, lvs
  6. Rebuild: pvcreate → vgcreate → lvcreate → mkfs → mount → fstab entry
  7. Verify rebuild with mount -a and df -Th
After teardown: all LVM commands return empty. After rebuild: functional LV with mounted filesystem.
Practice this cycle until it is second nature. You must execute the full LVM workflow from memory on the exam.

Section 2: Manage and Configure the Virtual File System

Labs 21–35 · /proc, /sys, tmpfs, devtmpfs, mount options, overlayfs, quotas

Lab 21: Explore /proc Filesystem

Understand /proc pseudo-filesystem and extract system info.

  1. Run cat /proc/meminfo — memory statistics
  2. Run cat /proc/cpuinfo — CPU details per core
  3. Run cat /proc/partitions — kernel-registered partitions
  4. Run cat /proc/mounts — all currently mounted filesystems
  5. Run cat /proc/filesystems — supported filesystem types
Explain what each file reveals about system hardware and configuration.

Lab 22: Explore /sys Filesystem

Navigate /sys to query hardware info.

  1. List block devices: ls /sys/block/
  2. Check I/O scheduler: cat /sys/block/sda/queue/scheduler
  3. Check disk size in sectors: cat /sys/block/sda/size
  4. Explore filesystem info: ls /sys/fs/
I/O scheduler shows active scheduler in brackets. Multiply sectors by 512 for bytes.

Lab 23: Modify Kernel Parameters via /proc/sys

Read/modify kernel tunables, make persistent.

  1. Read current: cat /proc/sys/vm/swappiness
  2. Modify live: echo 10 > /proc/sys/vm/swappiness
  3. Verify: sysctl vm.swappiness
  4. Make persistent: add vm.swappiness=10 to /etc/sysctl.conf or /etc/sysctl.d/
  5. Apply: sysctl -p
sysctl vm.swappiness returns 10 before and after reboot.

Lab 24: Working with tmpfs

Create RAM-backed filesystem.

  1. Create mount point: mkdir /mnt/ramdisk
  2. Mount tmpfs: mount -t tmpfs -o size=256M tmpfs /mnt/ramdisk
  3. Write test file: dd if=/dev/urandom of=/mnt/ramdisk/test bs=1M count=50
  4. Check RAM usage: free -h
  5. Add to fstab: tmpfs /mnt/ramdisk tmpfs size=256M 0 0
df -Th shows tmpfs with 256 MB. Files are lost after reboot.

Lab 25: Working with devtmpfs and udev

Understand /dev device management and write a udev rule.

  1. List devices: ls -la /dev/sd*
  2. Query device info: udevadm info /dev/sdb
  3. Full attribute walk: udevadm info -a /dev/sdb
  4. Create udev rule in /etc/udev/rules.d/99-custom.rules for a symlink
  5. Reload: udevadm control --reload-rules && udevadm trigger
ls -la /dev/my_data_disk shows symlink to /dev/sdb.

Lab 26: Explore Mount Namespaces

Use findmnt and other tools to explore the mount tree.

  1. Tree view: findmnt
  2. Compare with mount and cat /proc/mounts
  3. Filter by type: findmnt -t ext4
  4. List view: findmnt -l
  5. Check specific mount: findmnt /mnt/projects
findmnt presents tree view. -t filter works correctly.

Lab 27: Configure Mount Options

Mount with security options and test behavior.

  1. Mount with restrictions: mount -o noexec,nosuid,nodev /dev/sdb1 /mnt/secure
  2. Copy a script and try to execute (fails with noexec)
  3. Remount with exec: mount -o remount,exec /mnt/secure
  4. Re-execute the script (succeeds)
mount output shows options changing between mounts.

Lab 28: Bind Mounts

Create bind mount mirroring a directory.

  1. Create target: mkdir /mnt/log-mirror
  2. Bind mount: mount --bind /var/log /mnt/log-mirror
  3. Verify identical content in both paths
  4. Add to fstab: /var/log /mnt/log-mirror none bind 0 0
Both paths show identical content. Creating file in one is visible in other.

Lab 29: Read-Only and Remount Operations

Mount read-only and practice remounting.

  1. Mount read-only: mount -o ro /dev/sdb1 /mnt/readonly
  2. Attempt write (fails with read-only error)
  3. Remount read-write: mount -o remount,rw /mnt/readonly
  4. Write succeeds after remount
mount output shows ro or rw as expected.

Lab 30: Overlay Filesystem (overlayfs)

Create overlay filesystem combining layers.

  1. Create directories: mkdir -p /tmp/overlay/{lower,upper,work,merged}
  2. Populate lower with test files
  3. Mount overlay: mount -t overlay overlay -o lowerdir=lower,upperdir=upper,workdir=work merged
  4. Read merged content (lower files visible)
  5. Write new content (goes to upper dir)
  6. Verify lower dir unchanged
Merged shows files from both layers. New writes go to upper. Lower unchanged.

Lab 31: Loop Devices and File-Backed Filesystems

Create filesystem inside a regular file.

  1. Create image: dd if=/dev/zero of=/tmp/disk.img bs=1M count=500
  2. Associate loop device: losetup /dev/loop0 /tmp/disk.img
  3. Create filesystem: mkfs.ext4 /dev/loop0
  4. Mount: mount /dev/loop0 /mnt/loop
  5. Write test data to verify
losetup -a shows association. df shows ~500 MB ext4.

Lab 32: Explore /proc/self and /proc/PID

Examine per-process info through /proc.

  1. View own mounts: cat /proc/self/mounts
  2. List own file descriptors: ls -la /proc/self/fd
  3. Start background process, note PID: sleep 600 & echo $!
  4. Examine: cat /proc/PID/status and cat /proc/PID/cmdline
  5. Check I/O stats: cat /proc/PID/io
/proc/self refers to the calling process. /proc/PID/io shows read/write bytes.

Lab 33: Configure Filesystem Quotas (User)

Enable per-user disk quotas on ext4.

  1. Add usrquota mount option, remount the filesystem
  2. Initialize quota database: quotacheck -cum /mnt/projects
  3. Enable quotas: quotaon /mnt/projects
  4. Set limits: edquota testuser (soft 50 MB, hard 100 MB)
  5. Test exceeding soft limit (warning)
  6. Test exceeding hard limit (write fails)
repquota /mnt/projects shows usage. quota testuser shows limits.

Lab 34: Configure Filesystem Quotas (Group)

Enable per-group disk quotas.

  1. Add grpquota mount option, remount
  2. Initialize: quotacheck -cgm /mnt/projects
  3. Enable: quotaon /mnt/projects
  4. Set group limits: edquota -g devteam
repquota -g /mnt/projects shows group reports.

Lab 35: Persistent Mount Configuration Audit

Audit /etc/fstab and recover from a bad entry.

  1. Review fstab, verify UUIDs match blkid output
  2. Validate: findmnt --verify
  3. Test: mount -a
  4. Add bad entry with non-existent UUID
  5. Reboot, observe boot failure (drops to emergency mode)
  6. Fix in emergency mode, reboot
System boots after fix. findmnt --verify returns clean.
Always add nofail option to non-critical mounts. This prevents boot failures if the device is missing.

Section 3: Create, Manage, and Troubleshoot Filesystems

Labs 36–55 · ext4, XFS, Btrfs, VFAT, labels, UUIDs, tune, repair, resize, quotas, troubleshooting

Lab 36: Create ext4 Filesystem

Create ext4 with custom label and reserved blocks.

  1. Create filesystem: mkfs.ext4 -L mydata -m 2 /dev/sdb1
tune2fs -l confirms label and 2% reserved. blkid shows LABEL and UUID.

Lab 37: Create XFS Filesystem

Create XFS and inspect metadata.

  1. Create: mkfs.xfs -L xfsdata /dev/sdb2
  2. Mount and inspect
  3. View metadata: xfs_info /mnt/xfsdata
xfs_info shows block size, log section, data section.

Lab 38: Create Btrfs Filesystem

Create Btrfs, subvolumes, and snapshots.

  1. Create: mkfs.btrfs -L btrfstest /dev/sdc1
  2. Mount the filesystem
  3. Create subvolume: btrfs subvolume create /mnt/btrfs/data
  4. Write data to subvolume
  5. Create read-only snapshot: btrfs subvolume snapshot -r /mnt/btrfs/data /mnt/btrfs/data-snap
  6. List subvolumes: btrfs subvolume list /mnt/btrfs
Both subvolume and snapshot are listed.

Lab 39: Create VFAT/FAT32 Filesystem

Create FAT32 for USB/EFI scenarios.

  1. Create: mkfs.vfat -F 32 /dev/sdd1
  2. Mount the filesystem
  3. Note that Linux permissions are not supported
df -Th shows "vfat".

Lab 40: Filesystem Labels and UUIDs

Set/modify labels, mount by UUID/LABEL.

  1. Set ext4 label: e2label /dev/sdb1 mydata
  2. Set XFS label: xfs_admin -L xfsdata /dev/sdb2
  3. Display UUIDs: blkid
  4. Mount by UUID in fstab: UUID=xxxx /mnt/data ext4 defaults 0 2
  5. Mount by LABEL in fstab: LABEL=mydata /mnt/data ext4 defaults 0 2
mount -a succeeds for both methods.
UUIDs are preferred over device paths because they persist across reboots. LABELs must be unique across the system.

Lab 41: Tune ext4 Filesystem Parameters

Modify ext4 parameters with tune2fs.

  1. Set max mount count: tune2fs -c 30 /dev/sdb1
  2. Set check interval: tune2fs -i 90d /dev/sdb1
  3. Reduce reserved blocks: tune2fs -m 1 /dev/sdb1
  4. Set reserved user: tune2fs -u testuser /dev/sdb1
  5. Force check at next boot
tune2fs -l /dev/sdb1 shows all changes applied.

Lab 42: Check and Repair ext4 Filesystem

Force check and practice superblock recovery.

  1. Unmount the filesystem
  2. Force check: e2fsck -f /dev/sdb1
  3. Find backup superblocks: dumpe2fs /dev/sdb1 | grep superblock
  4. Corrupt superblock with dd (test only!)
  5. Attempt normal check (fails)
  6. Repair with backup superblock: e2fsck -b 32768 /dev/sdb1
Filesystem mounts after repair, data intact.
Corrupting superblock destroys critical metadata. Only do this on a test filesystem you can afford to lose.

Lab 43: Check and Repair XFS Filesystem

Use xfs_repair to check and fix XFS.

  1. Unmount the filesystem
  2. Dry run: xfs_repair -n /dev/sdb2
  3. Repair: xfs_repair /dev/sdb2
  4. If dirty log: xfs_repair -L /dev/sdb2 (last resort — zeroes log)
xfs_repair completes clean. Remount succeeds.
XFS has no traditional fsck. xfs_repair is the equivalent. xfs_check is deprecated.

Lab 44: Resize ext4 Filesystem

Shrink and grow ext4 with resize2fs.

  1. Unmount the filesystem
  2. Force check: e2fsck -f /dev/sdb1
  3. Shrink: resize2fs /dev/sdb1 500M
  4. Remount, verify ~500 MB
  5. Grow online: resize2fs /dev/sdb1 (no size = use all available space)
  6. Verify full partition size
After shrink ~500 MB, after grow full partition size.

Lab 45: Grow XFS Filesystem

Extend XFS after expanding underlying storage.

  1. Extend LV: lvextend -L +500M /dev/vg_storage/lv_backups
  2. Grow XFS: xfs_growfs /mnt/backups
df -Th shows increased size.
XFS can only be grown, never shrunk. This is a one-way operation.

Lab 46: Mount Filesystems Manually

Practice mounting by device, UUID, and LABEL.

  1. Mount by device: mount /dev/sdb1 /mnt/data
  2. Mount by UUID: mount UUID=xxxx /mnt/data
  3. Mount by LABEL: mount LABEL=mydata /mnt/data
  4. Mount with explicit type: mount -t ext4 /dev/sdb1 /mnt/data
Each method mounts successfully.

Lab 47: Configure /etc/fstab Entries

Write correct fstab entries, understand all 6 fields.

  1. Add entries for ext4, xfs, swap using UUID
  2. Document all 6 fields: Device, Mount Point, Type, Options, Dump, Pass
  3. Test: mount -a
mount -a produces no errors. All entries visible in findmnt.

Lab 48: Use systemd Mount Units

Create .mount unit file as alternative to fstab.

  1. Create /etc/systemd/system/mnt-data.mount
  2. Add [Unit], [Mount], [Install] sections
  3. Reload: systemctl daemon-reload
  4. Enable and start: systemctl enable --now mnt-data.mount
systemctl status mnt-data.mount shows "active (mounted)". findmnt shows mount.

Lab 49: Troubleshoot Boot Failures from Bad fstab

Break fstab, experience failure, recover.

  1. Add bad UUID entry without nofail
  2. Reboot (drops to emergency mode)
  3. Enter root password
  4. Remount root read-write: mount -o remount,rw /
  5. Fix /etc/fstab
  6. Reboot to verify
System boots normally after fix. Take a snapshot before this lab!

Lab 50: Recover Data from Damaged Filesystem

Practice ext4 recovery with backup superblocks.

  1. Create and populate ext4 filesystem
  2. Find backups: dumpe2fs /dev/sdb1 | grep superblock
  3. Corrupt primary superblock with dd
  4. Recover: e2fsck -b 32768 /dev/sdb1
  5. Mount and verify data integrity
Filesystem mounts, most/all files recoverable.

Lab 51: Working with Disk Usage Tools

Analyze disk space with multiple tools.

  1. Check directory size: du -sh /var/log
  2. Breakdown: du --max-depth=1 -h /var
  3. Interactive: ncdu /var
  4. Inode usage: df -ih
  5. Find large files: find / -xdev -size +100M
Can identify largest directories and files using each tool.

Lab 52: Managing Inodes

Understand inode exhaustion.

  1. Create filesystem with limited inodes: mkfs.ext4 -N 1000 /dev/sdb1
  2. Create many small files in a loop
  3. Observe "No space left on device" error with free space remaining
  4. Compare df -ih (100% inode usage) vs df -h (free space available)
df -h shows free space but df -i shows 100% inode usage.
XFS allocates inodes dynamically, making inode exhaustion much less likely than on ext4.

Lab 53: File Attributes and Extended Attributes

Use chattr, lsattr, setfattr, getfattr.

  1. Set immutable: chattr +i /mnt/data/important.txt
  2. Verify: lsattr /mnt/data/important.txt
  3. Try delete (fails — Operation not permitted)
  4. Remove immutable: chattr -i /mnt/data/important.txt
  5. Set append-only: chattr +a /mnt/data/logfile.txt
  6. Set extended attribute: setfattr -n user.department -v "engineering" /mnt/data/file.txt
  7. Read: getfattr -d /mnt/data/file.txt
lsattr shows flags. getfattr returns stored value.

Lab 54: Filesystem Freeze and Unfreeze

Freeze filesystem for consistent snapshots.

  1. Freeze: fsfreeze -f /mnt/data
  2. Attempt write in another terminal (hangs)
  3. Take LVM snapshot or backup while frozen
  4. Unfreeze: fsfreeze -u /mnt/data
  5. Hanging write completes
While frozen, I/O blocks. After unfreeze, I/O resumes.
fsfreeze is the filesystem-agnostic alternative to xfs_freeze. Works on ext4, XFS, and more.

Lab 55: Comprehensive Filesystem Troubleshooting Scenario

Diagnose and fix multiple filesystem problems in sequence.

  1. Problem 1: Wrong fstab UUID — check journalctl -b, compare blkid, fix UUID
  2. Problem 2: Full filesystem — df -h, du, find and remove large files
  3. Problem 3: Inode exhaustion — df -i, find dirs with many small files, clean up
  4. Problem 4: Corrupted superblock — e2fsck -b 32768 to repair
All four problems resolved, all filesystems mounted and working.

Section 4: Use Remote Filesystems and Network Block Devices

Labs 56–72 · NFS, CIFS/Samba, SSHFS, iSCSI, NBD, Kerberos, Troubleshooting

Labs 56–72 require two VMs or a single VM using loopback (127.0.0.1) for client and server roles.

Lab 56: Install and Configure NFS Server

Set up NFS server and export a directory.

  1. Install: apt install nfs-kernel-server (or dnf install nfs-utils)
  2. Create share: mkdir -p /srv/nfs/shared && chmod 777 /srv/nfs/shared
  3. Configure /etc/exports: /srv/nfs/shared *(rw,sync,no_subtree_check)
  4. Apply: exportfs -ra
  5. Enable: systemctl enable --now nfs-server
exportfs -v confirms export. showmount -e localhost lists share.

Lab 57: Mount NFS Share (Manual)

Manually mount NFS export.

  1. Install client: apt install nfs-common
  2. Create mount point: mkdir /mnt/nfs
  3. Mount: mount -t nfs server:/srv/nfs/shared /mnt/nfs
  4. Test read/write operations
df -Th shows nfs4 type.

Lab 58: Persistent NFS Mount via fstab

Configure persistent NFS mount.

  1. Add fstab entry: server:/srv/nfs/shared /mnt/nfs nfs4 _netdev,defaults 0 0
  2. Test: mount -a
  3. Reboot and verify mount persists
findmnt shows NFS after reboot.
_netdev tells systemd to wait for network before mounting. Always use for network mounts.

Section 5: Configure and Manage Swap Space

Labs 73–82 · Swap partitions, swap files, LVM swap, priorities, encryption, troubleshooting

Lab 73: Check Current Swap Configuration

Identify active swap sources.

  1. Show swap details: swapon --show
  2. Check memory: free -h
  3. Kernel view: cat /proc/swaps
All three commands agree on total swap size and sources.

Lab 74: Create Swap Partition

Create and activate swap partition.

  1. Create partition with fdisk, set type 82 (Linux swap)
  2. Initialize: mkswap /dev/sde1
  3. Activate: swapon /dev/sde1
swapon --show lists it. free -h shows increased swap.

Lab 75: Create Swap File

Create swap file alternative to partition.

  1. Create: dd if=/dev/zero of=/swapfile bs=1M count=512
  2. Set permissions: chmod 600 /swapfile
  3. Initialize: mkswap /swapfile
  4. Activate: swapon /swapfile
  5. Add to fstab: /swapfile none swap sw 0 0
swapon --show lists swap file. Persists after reboot.
Use dd, not fallocate, for swap files on older kernels. dd is the safer approach.

Lab 76: Remove Swap Space

Safely remove swap sources.

  1. Deactivate: swapoff /swapfile
  2. Remove fstab entry
  3. Delete file: rm /swapfile
  4. Deactivate partition: swapoff /dev/sde1
swapon --show no longer lists removed sources.

Lab 77: Set Swap Priority

Configure multiple swap sources with priorities.

  1. Create two swap sources (file + partition)
  2. Activate with priorities: swapon -p 10 /dev/sde1 and swapon -p 5 /swapfile
  3. Or use pri= in fstab entries
swapon --show displays priorities. Highest priority is used first. Equal priority = round-robin.

Lab 78: Create Swap on LVM

Dedicated LVM LV for swap.

  1. Create LV: lvcreate -n lv_swap -L 1G vg_storage
  2. Initialize: mkswap /dev/vg_storage/lv_swap
  3. Activate: swapon /dev/vg_storage/lv_swap
  4. Add to fstab
swapon --show lists LVM swap. lvs shows swap LV.

Lab 79: Resize Swap on LVM

Resize LVM swap volume.

  1. Deactivate: swapoff /dev/vg_storage/lv_swap
  2. Extend: lvextend -L +512M /dev/vg_storage/lv_swap
  3. Re-initialize: mkswap /dev/vg_storage/lv_swap
  4. Activate: swapon /dev/vg_storage/lv_swap
swapon --show shows new 1.5 GB size.
You must run mkswap again after resizing a swap LV. This is a frequently missed step.

Lab 80: Tune Swappiness

Adjust kernel swappiness parameter.

  1. Read current: cat /proc/sys/vm/swappiness (default 60)
  2. Set live: sysctl vm.swappiness=10
  3. Make persistent: add to /etc/sysctl.d/99-swap.conf
  4. Apply: sysctl -p /etc/sysctl.d/99-swap.conf
sysctl vm.swappiness returns 10.
0 = avoid swapping except under extreme pressure. 100 = aggressive swapping. Servers: 10–30. Desktop: default 60.

Lab 81: Encrypted Swap

Configure encrypted swap with dm-crypt random key.

  1. Install: apt install cryptsetup
  2. Add /etc/crypttab entry: crypt_swap /dev/sde1 /dev/urandom swap
  3. Update fstab: /dev/mapper/crypt_swap none swap sw 0 0
  4. Reboot to activate
swapon --show shows dm device. cryptsetup status crypt_swap confirms encryption.
/dev/urandom key = new random key each boot. Swap is unrecoverable after reboot. Do not use for hibernate.

Lab 82: Swap Troubleshooting Scenarios

Diagnose common swap problems.

  1. Swap not activating: check fstab syntax, verify UUID with blkid, check journalctl -b
  2. Wrong UUID: update fstab with correct UUID from blkid
  3. Full swap + OOM: free -h, dmesg | grep oom, find memory-hungry processes with top
Each scenario resolved systematically.

System-wide I/O and memory statistics.

  1. Run: vmstat 1 10

Lab 97: Monitor with blktrace and blkparse

Deep I/O tracing at the block layer.

  1. Install: apt install blktrace

  3. Generate I/O activity
  4. Stop tracing (kill blktrace)
  5. Parse: blkparse -i trace
  6. Review detailed I/O events with timestamps
Trace output shows detailed I/O events with timestamps and operation types.

Lab 98: Benchmark with fio

Structured storage benchmarks with fio.

  1. Install: apt install fio
  2. Sequential read: fio --name=seqread --rw=read --bs=1M --size=500M --numjobs=1
  3. Sequential write: fio --name=seqwrite --rw=write --bs=1M --size=500M
  4. Random read: fio --name=randread --rw=randread --bs=4k --size=500M
  5. Random write: fio --name=randwrite --rw=randwrite --bs=4k --size=500M
MetricDescription
IOPSI/O operations per second
BWBandwidth (throughput)
latLatency (avg, p99, max)
Sequential operations are faster than random. Compare ext4 vs XFS results.

Lab 99: Benchmark with dd

Quick speed measurements with dd.

  1. Write test: dd if=/dev/zero of=testfile bs=1M count=1024 conv=fdatasync
  2. Clear cache: echo 3 > /proc/sys/vm/drop_caches
  3. Read test: dd if=testfile of=/dev/null bs=1M
  4. Cleanup: rm testfile
dd reports transfer rate in MB/s.
conv=fdatasync forces physical write. oflag=direct bypasses page cache. Always clear caches between tests.

  5. Collect sar -d data during test
  6. Analyze: highest %util, top I/O processes, swap activity, IOPS/latency
  7. Write performance report with findings and recommendations
Correlate metrics across all tools. Report includes workload type, IOPS, throughput, latency, utilization, and optimization recommendations.
