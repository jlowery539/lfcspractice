Section 1: Creating and Managing Local Users and Groups

This section covers the foundational skills of creating, modifying, and deleting local user and group accounts on a Linux system. You will work extensively with useradd, usermod, userdel, groupadd, groupmod, groupdel, passwd, chage, and gpasswd. Understanding the files that underpin these commands — /etc/passwd, /etc/shadow, /etc/group, and /etc/gshadow — is critical for the LFCS exam. Every lab in this section can be completed on a single VM with root access.

Lab 1: Creating a Basic User with useradd

Objective: Create a new local user using useradd with default settings and verify the account was created correctly in the system files and on disk.

Tasks:

  1. Create a new user named labuser1 using the useradd command.
  2. Verify the user entry exists in /etc/passwd.
  3. Check that a home directory was created under /home/.
  4. Verify the user’s default group was created in /etc/group.

Solution:

# Create the user
sudo useradd labuser1

# Verify the entry in /etc/passwd
grep labuser1 /etc/passwd
# Expected output:
# labuser1:x:1001:1001::/home/labuser1:/bin/bash

# Check that the home directory exists
ls -la /home/labuser1
# Expected output: directory listing showing .bash_profile, .bashrc, etc.

# Verify the user's primary group
grep labuser1 /etc/group
# Expected output:
# labuser1:x:1001:

Note

On CentOS/RHEL, useradd creates the home directory by default (CREATE_HOME yes in /etc/login.defs). On Ubuntu, you may need to use useradd -m to explicitly create the home directory.

Lab 7: Setting and Changing User Passwords

Objective: Set, change, expire, lock, and unlock user passwords using the passwd command.

Tasks:

  1. Set an initial password for labuser1.
  2. Force labuser1 to change the password at next login.
  3. Lock the labuser1 account.
  4. Unlock the labuser1 account.
  5. Verify the account status at each stage.

Solution:

# Set initial password (you will be prompted to enter it twice)
sudo passwd labuser1

# Force password change on next login
sudo passwd -e labuser1
# Verify:
sudo chage -l labuser1
# "Password must be changed" or "Last password change: password must be changed"

# Lock the account
sudo passwd -l labuser1
# Verify:
sudo passwd -S labuser1
# Expected: labuser1 LK ... (LK = locked)

# Unlock the account
sudo passwd -u labuser1
# Verify:
sudo passwd -S labuser1
# Expected: labuser1 PS ... (PS = password set)

Warning

Locking an account with passwd -l prepends ! to the password hash in /etc/shadow. This prevents password-based authentication but does not prevent login via SSH keys. To fully disable an account, also set the shell to /usr/sbin/nologin.

Lab 8: Modifying User Properties with usermod

Objective: Change an existing user’s comment (GECOS), login shell, and supplementary group membership using usermod.

Tasks:

  1. Change the comment field for labuser1 to “Lab User One”.
  2. Change the default shell for labuser1 to /bin/bash.
  3. Add labuser1 to the wheel group as a supplementary group.
  4. Verify all changes.

Solution:

# Change GECOS/comment field
sudo usermod -c "Lab User One" labuser1

# Change shell
sudo usermod -s /bin/bash labuser1

# Add to supplementary group (use -aG to APPEND)
sudo usermod -aG wheel labuser1

# Verify all changes
grep labuser1 /etc/passwd
# Expected: labuser1:x:1001:1001:Lab User One:/home/labuser1:/bin/bash

id labuser1
# Expected: uid=1001(labuser1) gid=1001(labuser1) groups=1001(labuser1),10(wheel)

Warning

Always use -aG (append) when adding supplementary groups. Using -G alone replaces all existing supplementary groups, which can lock a user out of critical groups like wheel or sudo.

Lab 9: Deleting Users with userdel

Objective: Remove user accounts with and without deleting their home directories and mail spools.

Tasks:

  1. Delete tempuser1 while preserving the home directory.
  2. Create a new user tempuser2, then delete the account along with its home directory.
  3. Verify the deletions.

Solution:

# Delete user, preserve home directory
sudo userdel tempuser1
# Verify user is gone but home remains
grep tempuser1 /etc/passwd        # No output
ls -la /home/tempuser1            # Directory still exists

# Create and then fully delete a user
sudo useradd -m tempuser2
sudo userdel -r tempuser2
# Verify both user and home are gone
grep tempuser2 /etc/passwd        # No output
ls /home/tempuser2                # "No such file or directory"

Warning

Never use userdel -r on production accounts without first verifying the contents of the home directory. Consider backing up the home directory before deletion.

Lab 13: Modifying Group Properties with groupmod

  2. Change the GID of testteam to 2500.

# Change the GID
sudo groupmod -g 2500 testteam

Lab 16: Understanding /etc/shadow File Structure

Objective: Read and interpret password aging fields in /etc/shadow.

Tasks:

  1. View /etc/shadow as root.
  2. Identify all nine fields for the labuser1 entry.
  3. Explain how the “last changed” date is stored as days since epoch.

Solution:

# View shadow file (requires root)

# Example entry:
# labuser1:$6$rounds=...:19844:0:99999:7:::
#    |        |            |    | |     | |||
#    |        |            |    | |     | ||+-- Field 9: Reserved
#    |        |            |    | |     | |+-- Field 8: Expiration date (days since epoch)
#    |        |            |    | |     | +-- Field 7: Inactive period (days)
#    |        |            |    | |     +-- Field 6: Warning period (days before expiry)
#    |        |            |    | +-- Field 5: Maximum password age (days)
#    |        |            |    +-- Field 4: Minimum password age (days)
#    |        |            +-- Field 3: Last password change (days since Jan 1, 1970)
#    |        +-- Field 2: Encrypted password hash
#    +-- Field 1: Username

2Password HashEncrypted password ($6$ = SHA-512)

4Min AgeMinimum days between password changes
5Max AgeMaximum days before password must change
6WarningDays before expiry to warn the user
7InactiveDays after expiry before account is disabled
8ExpireAbsolute expiration date (days since epoch)
9ReservedReserved for future use

Lab 25: Batch User Creation with a Script

Objective: Automate user creation by reading from a CSV file using a bash script.

Tasks:

  1. Create a CSV file with fields: username, shell, group.
  2. Write a bash script that reads the CSV and creates each user.
  3. Verify all users were created.

Solution:

# Step 1: Create the CSV file
cat << 'EOF' > /tmp/users.csv
batchuser1,/bin/bash,devteam
batchuser2,/bin/bash,engineering
batchuser3,/bin/zsh,research
EOF

# Step 2: Create the script
cat << 'SCRIPT' > /tmp/create_users.sh
#!/bin/bash
# Batch user creation script
INPUT_FILE="$1"

if [ -z "$INPUT_FILE" ]; then
    echo "Usage: $0 <csv_file>"
    exit 1
fi

while IFS=',' read -r username shell group; do
    # Skip empty lines
    [ -z "$username" ] && continue

    # Check if user already exists
    if id "$username" &>/dev/null; then
        echo "User $username already exists. Skipping."
        continue
    fi

    # Create the user
    useradd -m -s "$shell" -G "$group" "$username"
    if [ $? -eq 0 ]; then

Section 4: Configuring and Managing ACLs

This section covers POSIX Access Control Lists (ACLs), which extend the traditional owner/group/other permission model. ACLs allow fine-grained access control where you need to grant different permission levels to multiple users and groups on the same file or directory. The LFCS exam tests your ability to use setfacl and getfacl, configure default (inheritable) ACLs, and troubleshoot ACL-related permission issues. These skills are essential for managing shared directories and collaborative environments.

Lab 65: Understanding the ACL Mask

Objective: Learn how the ACL mask limits effective permissions for named users, named groups, and the owning group.

Tasks:

  1. Set an ACL on a file.
  2. Observe the mask value.
  3. Modify the mask and see how effective permissions change.

Solution:

# View ACL with mask
getfacl /shared/masktest
# Expected:
# user::rw-
# user:labuser1:rwx
# group::r--
# mask::rwx              <-- mask allows full rwx

Objective: Write a bash script to audit and report ACL usage across a directory tree.

  2. Run getfacl on each file and output a formatted report.

# Create the audit script

#!/bin/bash
# ACL Audit Script — scans a directory tree for files with ACLs

TARGET="${1:-/shared}"
REPORT="/tmp/acl_audit_$(date +%Y%m%d_%H%M%S).txt"

echo "======================================" > "$REPORT"
echo "  ACL Audit Report"                     >> "$REPORT"
echo "  Directory: $TARGET"                   >> "$REPORT"

echo "======================================" >> "$REPORT"

COUNT=0

done < <(find "$TARGET" -print0 2>/dev/null)

echo "======================================" >> "$REPORT"
echo "  Total files/dirs with ACLs: $COUNT"   >> "$REPORT"
echo "======================================" >> "$REPORT"

echo "Audit complete. Report saved to: $REPORT"
echo "Total files/directories with ACLs: $COUNT"
SCRIPT

Lab 99: LDAP Account Management Automation Script

Objective: Write a bash script to automate common LDAP user management tasks: add, delete, reset password, and list users.

Tasks:

  1. Create a bash script with a menu for LDAP user operations.
  2. Implement functions for each operation using ldapadd, ldapdelete, ldappasswd, and ldapsearch.
  3. Test each operation.

Solution:

cat << 'SCRIPT' > /usr/local/bin/ldap-manager.sh
#!/bin/bash
# LDAP User Management Script
# Usage: ldap-manager.sh [add|delete|passwd|list]

BASE_DN="dc=example,dc=com"
PEOPLE_DN="ou=People,$BASE_DN"
ADMIN_DN="cn=admin,$BASE_DN"

add_user() {
    read -p "Username: " USERNAME
    read -p "First Name: " FIRST
    read -p "Last Name: " LAST
    read -p "UID Number: " UIDNUM
    read -p "GID Number: " GIDNUM
    PASS_HASH=$(slappasswd -s "$(read -sp 'Password: ' P; echo "$P")")
    echo ""

    cat << EOF | ldapadd -x -D "$ADMIN_DN" -W
dn: uid=$USERNAME,$PEOPLE_DN
objectClass: inetOrgPerson
objectClass: posixAccount
objectClass: shadowAccount
uid: $USERNAME
cn: $FIRST $LAST
sn: $LAST
givenName: $FIRST
uidNumber: $UIDNUM
gidNumber: $GIDNUM
homeDirectory: /home/$USERNAME
loginShell: /bin/bash
userPassword: $PASS_HASH
EOF
    echo "User $USERNAME added."
}

delete_user() {
    read -p "Username to delete: " USERNAME
    ldapdelete -x -D "$ADMIN_DN" -W "uid=$USERNAME,$PEOPLE_DN"
    echo "User $USERNAME deleted."
}

reset_password() {
    read -p "Username: " USERNAME
    ldappasswd -x -D "$ADMIN_DN" -W -S "uid=$USERNAME,$PEOPLE_DN"
    echo "Password reset for $USERNAME."
}

list_users() {
    echo "=== LDAP Users ==="
    ldapsearch -x -b "$PEOPLE_DN" "(objectClass=posixAccount)" \
        uid cn uidNumber gidNumber loginShell -LLL
}

case "$1" in
    add)     add_user ;;
    delete)  delete_user ;;
    passwd)  reset_password ;;
    list)    list_users ;;
    *)
        echo "Usage: $0 {add|delete|passwd|list}"
        echo "  add     - Add a new LDAP user"
        echo "  delete  - Delete an LDAP user"
        echo "  passwd  - Reset user password"
        echo "  list    - List all LDAP users"
        exit 1
        ;;
esac
SCRIPT

sudo chmod +x /usr/local/bin/ldap-manager.sh

# Test
sudo ldap-manager.sh list
# Expected: displays all LDAP users
